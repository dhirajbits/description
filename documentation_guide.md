# High level description

This describes the project at high level. Like the requirements, mini app, code structure, etc. of the project.

## About `requirements` folder

This folder contains the file that describes whats the requirement of the project. What the need it have to fullfill

#### About `user_stories.md` file

This file is inside of `requirements` folder and fullfills the purpose of this folder.
It contains user stories. This is the stories which is keet in mind when building the app.
This tell about how user what to use the app and what they expect from it.

#### About `function_requirements.md` file

This file also inside `requirements` folder and align with the purpose of it.
As the name implise, It contains the information about function requirements of the project.

## About `implementation` folder

The responsibility is to describe the implementations logic of "user stories" and "functional requirements"of the project. It makes the initial decisions of project structure.

Generally it describes about the required mini apps with their purpose. And also describe the tech stack and technologies that is decided to use.

#### About `mini_apps.md` file

This files describes the required mini app with the detailing of the their layer depth and purpose.

#### About `tech_stack.md` file

Describes about the tech stack and technologies (library and other dependencies) of the project.

## About `code` folder

Files of this folder describes how to write and structure the code. This gives direction of folder structure

#### About `general_coding_rules.md` file

This describes the general coding rule to follow. It overwrites the local rule because this is project specific.

#### About `standard_code_structure.md` file
